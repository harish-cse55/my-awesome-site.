Zero to Live: How I Architected this Website

H

CSE Engineer • Infrastructure & Android

"As a CSE engineer, I didn't want a drag-and-drop site. I wanted a deployment pipeline I could control."

Step 1: The Tech Stack Choice

I chose GitHub Pages for hosting. It's free, fast, and integrates directly with my git workflow. For styling, I used Tailwind CSS via CDN to keep the initial load time under 500ms.

Step 2: DNS & Domain Mapping

I purchased harishcodes.online. Mapping this required updating the A Records in my DNS provider to point to GitHub's global edge servers.

Hostinger A-Records Used:

185.199.108.153

185.199.109.153

185.199.110.153

185.199.111.153

Step 3: Solving the 404 & Directory Logic

The biggest challenge was the directory structure. I wanted a separate /blog subdirectory. Initially, I faced 404 errors because the index.html was in the wrong folder.

The Fix: Keeping the main portfolio at the root / and the blog index at /blog/.

root/
├── index.html     # Main Portfolio
├── CNAME          # harishcodes.online
└── blog/
    └── index.html # Blog Feed List

Step 4: SSL & Security

Once the DNS was verified, I enabled Enforce HTTPS. GitHub automatically generated a TLS certificate via Let's Encrypt, ensuring all data between my readers and the server is encrypted.
