Mapping a Custom Domain

H

Harish

CSE Engineer • Android Developer

Today marks a major milestone for my portfolio. I successfully moved away from the default GitHub sub-domain to my personal domain: harishcodes.online.

The Challenge: 404s and DNS Lag

Initially, I faced the dreaded 404 error. This usually happens for two reasons: a missing CNAME file in the repository or incorrect A-Records in the DNS provider (Hostinger, in my case).

The DNS Configuration

For an apex domain (the main domain without 'www'), GitHub recommends using four specific IP addresses. I had to go into my Hostinger panel, delete the default parking records, and add these:

Type Name Value
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153

Verifying the Setup

As a CSE engineer, I don't just wait for it to work; I verify it via the terminal. Using the dig command, I checked if the propagation was complete:

# Checking my new DNS setup
dig harishcodes.online

# Expected result: 4 A records pointing to GitHub IPs

The Final Piece: SSL & Root Files

The final hurdle was ensuring my index.html was in the root directory and not tucked away in a folder. Once GitHub's "Actions" finished building, I enabled Enforce HTTPS in settings. Now, the site is live, secure, and ready for my upcoming Android projects!
