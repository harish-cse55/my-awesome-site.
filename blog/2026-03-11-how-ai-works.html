How AI Actually "Thinks": The Logic of Tokens

H

Harish

CSE Engineer • Tech Blogger

Most people think AI "reads" sentences like humans do. But as engineers, we know computers only understand numbers. To bridge this gap, AI uses Tokenization.

1. What is a Token?

A token is the basic unit of text. It isn't always a full word; it can be a part of a word or punctuation. For example, "Engineering" might be split into "Engine" and "ering".

2. Semantic Embeddings & Vector Space

After tokenization, the AI maps that token into a high-dimensional Vector Space. Every token becomes a list of numbers (a vector). Similar meanings are placed closer together.

Vector Arithmetic

Because tokens are vectors, we can perform math on them:

Vector("King") - Vector("Man") + Vector("Woman") ≈ Vector("Queen")

3. The Probability Equation

The AI calculates the probability of the *next* token based on previous ones using the Transformer architecture.

The probability $P$ of a sequence is:

$$P(w_1, ..., w_n) = \prod_{i=1}^{n} P(w_i | w_1, ..., w_{i-1})$$

4. Key Engineering Features

  • Context Window: The "RAM" of the model (e.g., 128k tokens).
  • Attention: How the model focuses on specific words for context.
  • Temperature: Controls randomness (0.0 for logic, 1.0 for creativity).
